# ERC-7984 ‚Äî OpenZeppelin Confidential Contracts

ERC-7984 is the encrypted-balance fungible-token standard. Public total supply, **encrypted** per-account balances, **encrypted** transfer amounts. Built on the Zama Protocol.

```bash
npm i @openzeppelin/confidential-contracts
```

---

## Minimal token

```solidity
pragma solidity ^0.8.27;

import {ERC7984}      from "@openzeppelin/confidential-contracts/token/ERC7984/ERC7984.sol";
import {SepoliaConfig} from "@fhevm/solidity/config/ZamaConfig.sol";

contract MyToken is ERC7984, SepoliaConfig {
    constructor() ERC7984("Confidential USD", "cUSD", "https://meta.example/cusd.json") {}
}
```

That's it. The base contract handles encrypted balance bookkeeping, ACL hygiene on every state mutation, transfers, operator approvals, and disclosure helpers.

## Surface area

| Function | Returns | Purpose |
|---|---|---|
| `confidentialBalanceOf(address)` | `euint64` | Returns the handle; user-decrypts off-chain |
| `confidentialTotalSupply()` | `euint64` | Encrypted supply (public reveal optional) |
| `confidentialTransfer(to, externalEuint64 amt, bytes proof)` | `euint64` actually transferred | Sender ‚Üí recipient |
| `confidentialTransferFrom(from, to, externalEuint64 amt, bytes proof)` | as above | Operator-mediated |
| `setOperator(operator, untilTimestamp)` | ‚Äî | Replaces ERC-20 `approve`; operator gets time-bounded authority |
| `isOperator(holder, operator)` | bool | Operator check |
| `discloseEncryptedAmount(euint64 amount)` | ‚Äî | Owner publicly decrypts a chosen amount handle |
| `_mint(to, plainAmount)` (internal) | ‚Äî | Mint plaintext into encrypted balance |

The transfer functions return the **actual** amount transferred ‚Äî capped at the sender balance. If a user tries to send more than they have, the function does *not* revert; it transfers `min(balance, amount)` and the difference is silently zero. This is deliberate: a revert would leak whether the balance was sufficient.

## Operator pattern (replaces approve/allowance)

ERC-20's `approve(spender, amount)` reveals the allowance amount. ERC-7984 instead uses time-bounded operators:

```solidity
token.setOperator(spender, block.timestamp + 1 days);
token.confidentialTransferFrom(holder, recipient, encAmt, proof);
```

The operator can spend up to the **encrypted balance** during the window. There's no separate allowance amount ‚Äî control is by time.

## Wrapping ERC-20 ‚Üí ERC-7984

`ERC7984ERC20Wrapper` lets you bring an existing ERC-20 (USDC, DAI‚Ä¶) into the confidential world:

```solidity
pragma solidity ^0.8.27;

import {ERC7984}              from "@openzeppelin/confidential-contracts/token/ERC7984/ERC7984.sol";
import {ERC7984ERC20Wrapper}  from "@openzeppelin/confidential-contracts/token/ERC7984/extensions/ERC7984ERC20Wrapper.sol";
import {SepoliaConfig}        from "@fhevm/solidity/config/ZamaConfig.sol";
import {IERC20}               from "@openzeppelin/contracts/token/ERC20/IERC20.sol";

contract cUSDC is ERC7984ERC20Wrapper, SepoliaConfig {
    constructor(IERC20 underlying)
        ERC7984ERC20Wrapper(underlying)
        ERC7984("Confidential USDC", "cUSDC", "")
    {}
}
```

### Wrap (ERC-20 ‚Üí encrypted)

```solidity
function wrap(address account, uint256 amount) public returns (uint64);
```

Pulls `amount` of the underlying with `transferFrom`, mints encrypted balance to `account`. Plaintext input (the wrap amount is public ‚Äî that's expected; the *holder's running balance* is what stays private).

### Unwrap (encrypted ‚Üí ERC-20)

```solidity
function unwrap(address from, address to, externalEuint64 amount, bytes calldata proof) public;
```

Requires an oracle async decryption: the wrapper requests decryption of the unwrap amount, and on the callback transfers that many underlying tokens out. Two-tx settlement.

## Common integration patterns

### Encrypted payroll
Mint cUSD to employees, each holder user-decrypts to see their own salary, payroll admin never knows individual amounts.

### Confidential DEX leg
Users wrap USDC ‚Üí cUSDC, trade in encrypted pools, unwrap on exit. The pool sees only encrypted amounts.

### Private DAO treasury
Treasury holds cUSDC. Members vote on encrypted disbursement amounts; oracle reveal triggers the unwrap.

## ACL hygiene (the wrapper handles it for you)

`ERC7984` already calls `FHE.allowThis` and `FHE.allow(_, holder)` on every balance mutation. You inherit correct behaviour by extending the base. **Do not override `_update` without re-applying ACL** ‚Äî that's the most common way to break the standard.

## Gotchas

| Pitfall | Detail |
|---|---|
| Transfer "succeeds" but value is 0 | Sender had insufficient balance; intentional non-revert |
| `unwrap` doesn't release tokens immediately | Oracle async; tokens released in callback tx |
| Operator timestamp in the past | `setOperator` no-ops silently; check return / event |
| Holder calls `confidentialBalanceOf(other)` | Returns handle, but user-decrypt fails (ACL excludes them) ‚Äî that's the privacy guarantee |
| Total supply leak | `confidentialTotalSupply` is encrypted by default; only revealed via `disclose‚Ä¶` |
